Key Information

  • Type: Standalone L2 Worker
  • Phases: 7 (sequential workflow)
  • Input: topic (required), scope, goals, analyses
  • Output: 7 Markdown documents in docs/market/[topic]/
  • Time-box: 45-60 minutes total
  • Position: Before ln-210-epic-coordinator

Tools Used

  • WebSearch - Market data, competitors, trends, customer segments
  • mcp__Ref__ref_search_documentation() - Industry reports, whitepapers
  • mcp__context7__get-library-docs() - Technology adoption trends
  • Glob - Check existing docs/market/ research
  • Write - Save output documents

Output Documents

  • competitor_analysis.md - Top-10 competitors, feature matrix, positioning map
  • market_sizing.md - TAM/SAM/SOM calculations with methodology
  • trends.md - 5-7 key trends with impact assessment
  • customer_profile.md - JTBD framework, 2-3 personas
  • ocean_analysis.md - Blue/Red Ocean, ERRC grid, non-customers
  • prioritization.md - RICE/ICE scored feature list
  • summary.md - Executive summary with recommendations

Important Notes

  • Research done ONCE per topic, results inform Epic planning
  • All data must be sourced with dates (prefer 2024-2025)
  • Cross-reference key metrics (TAM, competitor count) with 2+ sources
  • Confidence levels (High/Medium/Low) assigned to all estimates
  • RICE/ICE scoring helps prioritize features for Epic scope
